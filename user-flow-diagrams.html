Admin User Journey

Step-by-step flow of what happens when an admin enters the RideShare SA platform.

1
Platform Entry
Admin visits ridesharex.co.za and sees the landing page
What happens:
  • Landing page loads with platform overview
  • Admin login button is prominently displayed
  • System checks for existing admin session
  • If no session, redirects to admin login page
  • If session exists, redirects to admin dashboard
↓
2
Admin Authentication
Admin enters credentials and completes security verification
Authentication Process:
  • Admin enters email and password
  • System validates credentials against admin database
  • Multi-factor authentication is triggered (SMS/Email)
  • Admin enters verification code
  • System creates secure admin session
  • Admin is redirected to dashboard
↓
3
Admin Dashboard
Admin sees comprehensive platform overview and key metrics
Dashboard Display:
  • Real-time platform statistics load
  • Active users, bookings, and revenue metrics shown
  • Recent activity feed displays
  • Alert notifications for critical issues
  • Quick action buttons for common tasks
  • Navigation menu with all admin functions
↓
4
User Management Actions
Admin reviews and manages user accounts and registrations
User Management Process:
  • Admin clicks on "User Management" section
  • System loads pending user registrations
  • Admin reviews user profiles and documents
  • Admin approves or rejects registrations
  • System sends notifications to users
  • User status is updated in database
↓
5
Vehicle Approval Process
Admin reviews and approves vehicle listings from hosts
Vehicle Approval Workflow:
  • Admin navigates to vehicle management
  • System shows pending vehicle listings
  • Admin reviews vehicle photos and details
  • Admin verifies vehicle documentation
  • Admin approves or requests changes
  • Vehicle becomes available for booking
↓
6
Financial Monitoring
Admin tracks platform revenue and financial performance
Financial Dashboard:
  • Admin accesses financial reports section
  • System displays revenue analytics
  • Transaction history and trends shown
  • Payout status to hosts displayed
  • Admin can process payouts manually
  • Financial alerts and notifications shown
↓
7
Safety & Security Monitoring
Admin monitors platform safety and handles security issues
Safety Management:
  • Admin reviews safety incident reports
  • System shows flagged content and users
  • Admin investigates security alerts
  • Admin takes action on violations
  • System logs all admin actions
  • Notifications sent to affected users
↓
8
Support & Communication
Admin handles customer support and platform communications
Support Operations:
  • Admin reviews support tickets
  • System shows live chat conversations
  • Admin responds to user inquiries
  • Admin escalates complex issues
  • System tracks response times
  • Admin sends platform announcements

Host User Journey

Step-by-step flow of what happens when a host enters the RideShare SA platform.

1
Platform Entry
Host visits ridesharex.co.za and sees the landing page
What happens:
  • Landing page loads with platform overview
  • "Become a Host" button is prominently displayed
  • System checks for existing user session
  • If logged in, redirects to host dashboard
  • If not logged in, shows registration options
↓
2
Host Registration
Host creates account and completes verification process
Registration Process:
  • Host clicks "Become a Host" button
  • Registration form loads with required fields
  • Host enters email, phone, and personal details
  • System sends verification email/SMS
  • Host verifies contact information
  • Account is created and pending approval
↓
3
Document Upload
Host uploads required documents for verification
Document Verification:
  • Host navigates to document upload section
  • System requests ID document, driver's license
  • Host uploads clear photos of documents
  • System validates document authenticity
  • Host sets up bank account for payouts
  • Documents are sent for admin review
↓
4
Vehicle Listing Creation
Host adds their first vehicle to the platform
Vehicle Listing Process:
  • Host clicks "Add Vehicle" button
  • Vehicle listing form loads with categories
  • Host uploads high-quality vehicle photos
  • Host enters vehicle details and specifications
  • Host sets pricing and availability calendar
  • Listing is submitted for admin approval
↓
5
Host Dashboard
Host sees their personalized dashboard with vehicle status
Dashboard Display:
  • Host dashboard loads with vehicle overview
  • Pending approvals and status updates shown
  • Earnings summary and payout information displayed
  • Recent activity and notifications shown
  • Quick action buttons for common tasks
  • Navigation menu with all host functions
↓
6
Booking Management
Host receives and manages booking requests
Booking Workflow:
  • Host receives booking request notification
  • System shows renter profile and rental details
  • Host reviews renter's history and ratings
  • Host approves or declines the booking
  • System sends confirmation to renter
  • Booking is added to host's calendar
↓
7
Guest Communication
Host communicates with renter before and during rental
Communication Process:
  • Host receives message from renter
  • System opens chat interface
  • Host provides pickup instructions and details
  • Host answers renter questions
  • System logs all communication
  • Host can send automated messages
↓
8
Earnings & Payouts
Host tracks earnings and receives payouts
Financial Process:
  • Host views earnings dashboard
  • System shows completed bookings and revenue
  • Host tracks pending and completed payouts
  • System processes automatic payouts
  • Host receives payment notifications
  • Financial reports are generated

Renter User Journey

Step-by-step flow of what happens when a renter enters the RideShare SA platform.

1
Platform Entry
Renter visits ridesharex.co.za and sees the landing page
What happens:
  • Landing page loads with featured vehicles
  • Search bar is prominently displayed
  • System shows available vehicles on map
  • If logged in, shows personalized recommendations
  • If not logged in, shows guest search options
↓
2
Vehicle Search
Renter searches for available vehicles by location and dates
Search Process:
  • Renter enters pickup location and dates
  • System loads available vehicles in area
  • Interactive map shows vehicle locations
  • Renter can filter by vehicle type and price
  • System displays search results with photos
  • Renter can compare multiple vehicles
↓
3
Vehicle Selection
Renter views vehicle details and selects preferred option
Selection Process:
  • Renter clicks on preferred vehicle
  • Vehicle details page loads with photos
  • System shows pricing, features, and host info
  • Renter reviews vehicle specifications
  • Renter checks availability calendar
  • Renter can contact host with questions
↓
4
Account Creation/Login
Renter creates account or logs in to proceed with booking
Authentication Process:
  • Renter clicks "Book Now" button
  • System prompts for login or registration
  • If new user, registration form loads
  • If existing user, login form appears
  • Renter completes identity verification
  • Account is verified and booking continues
↓
5
Booking Details
Renter confirms booking dates, times, and pickup details
Booking Configuration:
  • Renter selects pickup and return dates
  • System shows available time slots
  • Renter chooses pickup and return locations
  • System calculates total rental cost
  • Renter adds insurance and extras
  • Renter reviews all booking details
↓
6
Payment Processing
Renter completes payment and receives booking confirmation
Payment Process:
  • Renter selects payment method
  • System shows secure payment form
  • Renter enters payment details
  • System processes payment securely
  • Booking confirmation is generated
  • Renter receives confirmation email/SMS
↓
7
Vehicle Pickup
Renter picks up the vehicle at the designated location
Pickup Process:
  • Renter receives pickup instructions
  • Renter navigates to pickup location
  • Renter meets host or uses contactless pickup
  • Vehicle inspection is conducted
  • Renter receives keys and documents
  • Pickup is confirmed in the app
↓
8
Vehicle Return
Renter returns the vehicle and completes the rental
Return Process:
  • Renter returns to designated location
  • Host conducts vehicle inspection
  • Renter returns keys and documents
  • Any damage or issues are documented
  • Return is confirmed in the app
  • Rental period is officially completed
↓
9
Review & Feedback
Renter rates the experience and provides feedback
Feedback Process:
  • System prompts for review and rating
  • Renter rates host and vehicle experience
  • Renter writes detailed review
  • Feedback is submitted to platform
  • Renter earns loyalty points
  • Review is published for other users
